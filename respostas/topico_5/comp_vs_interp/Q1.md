# comp_vs_interp
## Q1
Compiladores fazem a tradução de um código-fonte inteiro para um código-alvo que pode ser interpretado por uma máquina. A cada mudança no código-fonte, este precisa ser re-compilado para poder ser executado. Já os interpretadores fazem a tradução linha a linha, sem a necessidade de ser emitido um código baixo-nível. A cada execução, o código é lido, traduzido e interpretado, sem a necessidade de haver uma compilação.  
As etapas entre compiladores e interpretadores são as mesmas até a análise semântica. Entretanto, essas etapas ocorrem em tempo de execução quando se trata de um interpretador, enquanto que para os compiladores elas ocorrem antes do código ser executado.  
Já as etapas de emissão de código e, consequentemente, a otimização só ocorrem para os compiladores, não havendo a sua necessidade nos interpretadores.
